在独立子会话中运行 Search Agent，执行面向 `inputs/library/**` 的证据检索，并返回报告定位信息供 Plan Agent 消费。

适用场景：
- 需要从资料库检索可追溯证据；
- 需要生成结构化搜索报告；
- 不用于直接撰写文章正文。

输入语义：
- `query`：本次检索的核心目标（要回答的问题）。
- `query_context`：检索背景、约束与补充条件（范围、时间、实体、排除项等）。

输出语义：
- `report_path`：搜索报告路径。Plan Agent 必须优先读取该文件，再基于报告中的引用使用证据。
- `sub_session_id`：搜索子会话 ID，用于审计检索过程。
- `assistant_message_id`：搜索子会话最终 assistant 消息 ID，用于追踪输出来源。

调用方义务：
- 拿到 `report_path` 后，必须先 `read` 报告再引用结论；
- 引用时应以报告中的 citations 为准，不能只凭工具 metadata 或主观总结。

失败语义：
- 若未返回 `report_path`，视为搜索未完成或未达标；
- 调用方应触发回退流程（如缩小问题、补充约束、或重新调用），而不是直接采用不完整结果。
